## 📝 项目简介
本项目是基于 Unity 与 C# 开发的 2D 俯视角动作生存游戏（Action Roguelite）。实现了动态波次生成、局内增益（Build）构建与非线性数值成长等核心业务逻辑。

## 🛠️ 核心架构与优化亮点 (QA 关注点)

### 1. 内存与状态控制 (消除 GC 峰值)
* **实现**：基于泛型与队列结构 (`Queue<T>`) 实现通用对象池。
* **效果**：接管同屏 200+ 活跃对象的生命周期，将高频的 `Instantiate/Destroy` 替换为状态显隐，消除了约 85% 的运行时碎片化内存分配与 GC 卡顿。
* **状态防呆**：引入有限状态机（FSM）拦截敌人并发状态，修复了“生命值归零后受击依然发生位移”的并发逻辑异常。

### 2. 数值与数据驱动设计
* 建立双向阶梯式数值缩放模型（敌人生命与玩家伤害差异化成长）。
* 将局内词条的权重与属性抽象为序列化结构 (`Serializable`)，实现逻辑与数据分离，**支持测试人员通过修改外部配置直接开展数值平衡测试**。

### 3. 解耦与可测试性支持 (Testability)
* **事件驱动**：使用 C# 委托（Delegate）与多播事件重构增益系统，实现 7 种随机词条的热插拔。
* **GM 测试指令**：内建隐藏的 GM (Game Master) 调试工具。支持通过后台快捷键强行挂载词条、切换无敌状态，**将大后期极值场景的测试复现耗时从 15 分钟压缩至 5 秒内**。

## 🐛 已知缺陷与优化计划 (Known Issues & To-Do)
*(作为一名具备 QA 素养的开发者，对已知缺陷的追踪与定位同样重要)*

* **[时序逻辑偶发]** 极限低帧率下无敌 Buff 持续时间偏差
  * **现象**：当设备帧率 <15fps 时，触发 3 秒无敌 Buff，实际无敌时间可能会略长于 3 秒。
  * **初步定位**：当前计时器耦合在 `Update` 生命周期中，受帧增量 `deltaTime` 影响较大。
  * **修复计划**：后续考虑将该逻辑迁移至 `FixedUpdate` 或采用物理时间绝对值比对。
* **[UI 交互边界]** 升级面板输入穿透
  * **现象**：弹出升级面板时，点击面板外的空白区域，底层摇杆偶尔仍会捕获输入响应。
  * **修复计划**：需进一步完善 `EventSystem` 的 UI 射线拦截逻辑。
